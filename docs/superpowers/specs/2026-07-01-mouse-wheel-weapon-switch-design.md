# 設計：マウスホイールで武器切り替え

- 日付: 2026-07-01
- ステータス: 承認済み（実装前）

## 目的

マウスホイールで所持武器を巡回切り替えできるようにする。既存の数字キー（1/2/3）による切り替えを補完する、より素早い操作手段。

## 非目標 (YAGNI)

- ホイールでの武器以外の操作（ズーム等）は対象外。
- 武器切り替えアニメーションやクイックスワップ演出の追加は対象外（既存の `Audio.switchWeapon()` / 速度ランプをそのまま利用）。

## データフロー（プロトコル変更なし）

ホイールイベントを `Input` シングルトンに蓄積し、`localInput.ts`（唯一の DOM 境界）が既存の絶対値 `weaponSlot: number | null` に変換する。`sysPlayer` は数字キーと全く同じ経路で適用する。

`PlayerInput` / スナップショット / `sysPlayer` は変更不要。ホイールはクライアントローカルで解決され、ワイヤーを渡るのは解決済みの `weaponSlot`（int）のみ。したがって **co-op はそのまま動作**する（ホスト権威・クライアントは意図のみ送信、という規約を維持）。

## 操作仕様

- **ホイール下（`deltaY > 0`）= 次の武器**（スロット番号を進める、ラップあり）。
- **ホイール上（`deltaY < 0`）= 前の武器**（スロット番号を戻す、ラップあり）。
- 巡回対象は **所持済み武器のみ**（`state.owned[id]`）。未所持スロットはスキップ。
- 端ではラップアラウンド（最後→最初、最初→最後）。
- 所持武器が 1 つ以下なら何もしない。
- 1 フレームに複数ノッチが溜まった場合はその数だけステップ（速いフリックで複数移動）。
- **数字キー優先**：同一フレームで数字キーによるスロット選択がある場合、ホイールは無視（競合回避）。
- ダウン中/観戦中はホイールを無視（`localInput` が既に早期 `emptyInput()` を返すため自然に成立）。

## 変更点

### 1. `game/input.ts`

- `wheel: 0`（number, ノッチ蓄積）を追加。
- canvas の `wheel` イベントで `this.wheel += Math.sign(e.deltaY)`、`e.preventDefault()`（ページスクロール抑制）。
- 既存の `blur` ハンドラで keys/firing をクリアするのと同じ箇所で `this.wheel = 0` にリセット。

### 2. 純粋ヘルパー（新規、ユニットテスト対象）

所持武器を巡回してスロットインデックスを解決する純粋関数を切り出す。

```
cycleWeaponSlot(order: readonly WeaponId[], owned: Record<WeaponId, boolean>, currentId: WeaponId, step: number): number | null
```

- `order` 内の所持済みインデックス列を作り、`currentId` の位置から `step` 分だけラップ移動した先の**絶対**スロットインデックスを返す。
- 所持が 1 つ以下、または移動先が現在と同じ場合は `null`。

配置先は既存のテスト方針（純粋・決定的コードのみ）に沿い、`game/systems/` または `game/data/` の適切なモジュール（実装時に既存の武器ヘルパー配置に合わせて決定。第一候補は `game/data/arsenal.ts` 近傍）。`*.test.ts` を co-locate。

### 3. `game/net/localInput.ts`

- 既存の数字キー解決の後、`weaponSlot` がまだ `null` の場合のみ `Input.wheel` を消費（読み取り後 0 リセット）。
- 非ゼロなら `cycleWeaponSlot(WEAPON_ORDER, state.owned, p.weapon, Input.wheel)` を呼び、結果を `weaponSlot` に代入。
- `Input.wheel` は毎フレーム必ず消費（数字キーが優先された場合でも溜め込まないよう 0 リセット）。

## テスト

- `cycleWeaponSlot` のユニットテスト：
  - 通常の次/前移動。
  - 端でのラップ（最後→最初、最初→最後）。
  - 未所持スロットのスキップ。
  - 所持 1 個以下 → `null`。
  - 複数ステップ（`step` が ±2 以上）。
- **操作感**（戦闘中のスクロール切り替えが気持ちいいか、数字キーとの競合が無いか）は feel-first 原則に従いプレイテストで確認。コンパイル/テスト通過だけでは done としない。
